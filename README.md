# Leafy – Premium Plant E-Commerce App

Leafy is a production-ready, client-side routed e-commerce platform designed for plant enthusiasts. The application features a crisp, modern UI focused on modular component architecture, robust global state management, and highly optimized routing behavior mimicking real-world web applications.

---

## 🚀 Key Features

- **Advanced Client-Side Routing:** Implements dynamic route parameters (`/product/:id`) and persistent nested layout structures using `react-router-dom`.
- **Global State Management:** Orchestrates cart interaction flows (add, update quantities, remove items) cleanly across diverse component layers utilizing **Redux Toolkit**.
- **Live Search Filtering:** Real-time, case-insensitive, partial-match search interface providing instant direct access to dynamic single product views.
- **Seamless UX Optimization:** Integrated custom programmatic scroll recovery configurations ensuring a smooth, fluid transit between sub-pages.

---

## 🎨 Design & UI Architecture

- **Visual Style:** Clean, high-contrast digital garden look using a deep emerald green framework against minimalist canvases.
- **Smooth Micro-interactions:** Fluid hover transitions, modular layout shells, and unified animation handling via `scroll-behavior: smooth`.
- **Typography Hierarchy:** Engineered with geometric sans-serif fonts to maximize clear technical reading structures for product grids and order breakdown layouts.

---

## 🛠️ Tech Stack & Tools

- **Frontend Framework:** React (Vite-powered)
- **Routing:** React Router DOM (Modern Data Architecture API)
- **State Engine:** Redux Toolkit & React-Redux
- **Styles:** Custom CSS3 Vanilla Layout Grids

---

## 📁 Project Structure

```text
src/
├── components/       # Reusable layout building blocks (Navbar, ScrollToTop, etc.)
├── data/             # Local structured database assets (products.js)
├── Pages/            # Primary view pages (Home, Products, SingleProducts, Cart)
├── store/            # Redux central configuration slice mechanics
├── App.jsx           # Main data router initialization config mapping
└── main.jsx          # Entry application rendering script execution
```
